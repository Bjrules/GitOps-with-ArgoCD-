# GitOps-with-ArgoCD-
Implementing GitOps Practices using ArgoCD 
It is Important to know the difference between Traditional CICD and GitOps and the Advantages of using GitOps over Deployment.

#### In this project, I successfully set up a complete CI (Continous Integration) pipeline using various DevOps tools on an AWS EKS cluster. I started by configuring essential services like Jenkins, Docker, SonarQube and Nexus to ensure seamless code integration, quality checks and artifact management.I also setup ArgoCD on EKS EKS cluster, for deploying applications on Kubernetes with it's pull-base technology for automated reconciliation of both Desired-state and Actual-state(EKS-Cluster). Thus, a seamless automation for deployment using Git as the "single point of truth" and that was how I built a robust and scalable environment for continuous delivery. 
> NB: GitOps with ArgoCD kind of automation wrks for both Infrastructure and Application.
